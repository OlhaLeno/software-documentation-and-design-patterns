# software-documentation-and-design-patterns

## Лабораторна робота №2: Система управління медіа-контентом (Netflix)
### Огляд проекту
Цей проект реалізує серверну частину системи для управління медіа-бібліотекою. Основна мета роботи — демонстрація навичок проектування складних систем з використанням тришарової архітектури (3-tier architecture), паттернів Dependency Injection (DI) та Inversion of Control (IoC).

Система дозволяє генерувати великі обсяги тестових даних (1000+ записів), імпортувати їх із зовнішніх джерел (CSV) та зберігати в реляційній базі даних з дотриманням принципів ООП та поліморфізму.

#### Архітектурні шари
Проект розділений на ізольовані модулі для забезпечення гнучкості та легкості тестування:

##### Core (Ядро):
Містить Domain Models (моделі предметної області) для SQLAlchemy.
Визначає Інтерфейси (абстрактні класи), які задають контракти для інших шарів.

##### DAL (Data Access Layer):
Реалізує роботу з базою даних через паттерн Repository.
Містить сервіс для читання сирих даних із файлової системи (CSV).

##### BLL (Business Logic Layer):
Обробляє дані, виконує конвертацію типів, валідацію та координує процес імпорту.
Повністю ізольований від конкретних технологій БД завдяки залежності лише від інтерфейсів.

##### PL (Presentation Layer):
Flask-додаток, що надає HTTP-інтерфейс для запуску бізнес-процесів.


#### Технологічний стек
Мова: Python 3.10+

Фреймворк: Flask

ORM: SQLAlchemy (SQLite)

Генерація даних: Faker Library

Архітектура: IoC / Dependency Injection


##### Інструкція з розгортання та запуску
1) Встановлення залежностей:

Середовище Python

Переконайтеся, що у вас встановлені необхідні бібліотеки:

pip install flask flask-sqlalchemy faker


3) Генерація тестових даних:

Модуль generator.py

Запустіть скрипт для створення CSV-файлу з 1000+ реалістичними записами:

python generator.py 1000


5) Запуск сервера:

Модуль app.py

Ініціалізуйте базу даних та запустіть Flask-сервер:

python app.py

Після запуску буде створено файл netflix.db.


7) Виконання імпорту:

HTTP Endpoint

Відкрийте браузер і перейдіть за адресою http://127.0.0.1:5000/import. Система виконає повний цикл імпорту даних із CSV до БД.

Обробка помилок: Система стійка до некоректних форматів дат або числових значень у вхідному файлі.

Оптимізація для Excel: Згенерований CSV містить технічні заголовки для коректного відображення в табличних редакторах без втрати можливості автоматичного парсингу.
