# Лабораторна робота №3: Паттерн MVC та CRUD операції

**Тема:** Система управління медіа-контентом (Netflix).

Цей проект є веб-додатком, побудованим на мікрофреймворку **Flask**, який демонструє практичне застосування архітектурного паттерну **MVC (Model-View-Controller)** та реалізацію базових операцій **CRUD** (Створення, Читання, Оновлення, Видалення) для роботи з базою даних SQLite за допомогою SQLAlchemy.

## Основний функціонал (Features)

* **Перегляд каталогу (Read):** Відображення списку всіх фільмів та серіалів у вигляді зручної таблиці з підтримкою детального перегляду (модальні вікна).
* **Створення контенту (Create):** Додавання нових записів (Фільм або Серіал) через веб-форму з динамічними полями.
* **Редагування (Update):** Зміна будь-яких параметрів існуючого контенту (назва, рейтинг, жанр, режисер, опис тощо).
* **Видалення (Delete):** Видалення контенту з бази даних із підтвердженням дії.
* **Імпорт даних:** Можливість автоматичного заповнення бази даних з CSV-файлу, згенерованого за допомогою скрипта.
* **Сучасний UI:** Інтерфейс стилізовано під платформу Netflix з використанням **Bootstrap 5**.

## Структура проекту (MVC)

Проект дотримується розділення логіки на шари:

* **Model (`core/models.py`):** Описує структуру даних (сутність `Content` та її спадкоємці `Movie`, `Serial`), взаємодіє з SQLite через SQLAlchemy.
* **View (`templates/` та `static/`):** HTML-шаблони (Jinja2), які відповідають за візуалізацію даних (список, форми, модальні вікна).
* **Controller (`routes.py`):** Обробляє HTTP-запити, керує маршрутизацією (Blueprints) та передає дані між Моделлю і Представленням.
* **BLL & DAL (`bll/`, `dal/`):** Класи бізнес-логіки та доступу до даних, що інкапсулюють роботу з репозиторіями та файлами.
* **`app.py`:** Точка входу в додаток, конфігурація та запуск сервера.

## Вимоги та встановлення

Для запуску проекту потрібен **Python 3.8+**.

1. **Клонуйте репозиторій або завантажте файли проекту.**
2. **Встановіть необхідні залежності:**
   Відкрийте термінал у папці проекту та виконайте:
   ```bash
   pip install flask flask-sqlalchemy faker
   ```

## Запуск проекту

1. **Згенеруйте тестові дані (Опціонально):**
   Щоб наповнити базу реалістичними даними англійською мовою, запустіть генератор (наприклад, для 50 записів):
   ```bash
   python generator.py 50
   ```
   *Це створить файл `data.csv` у корені проекту.*

2. **Запустіть Flask-сервер:**
   ```bash
   python app.py
   ```
   *База даних `netflix.db` створиться автоматично при першому запуску.*

3. **Відкрийте додаток у браузері:**
   Перейдіть за адресою: `http://127.0.0.1:5000/`

4. **Імпортуйте дані:**
   На головній сторінці натисніть кнопку **"Import from CSV"**, щоб завантажити згенеровані дані у базу.
